## Overview

This framework automatically scans API specifications (OpenAPI/Swagger), generates
comprehensive Java test suites using an LLM, and executes them with full Allure reporting
and GitHub Actions CI/CD integration.

**Key capabilities:**
- AI-powered test generation from OpenAPI specs (Claude / GPT)
- Shared library reused across all API capstone projects
- Self-healing test detection for outdated tests
- Parallel execution via TestNG
- Allure HTML reports published to GitHub Pages on every main branch push

## Quick start

### Prerequisites
- Java 17+
- Maven 3.9+

### Run tests locally

```bash
# Clone the repo
git clone https://github.com/njmarshall/ai-test-automation.git
cd ai-test-automation

# Build shared library
mvn install -pl shared -am -DskipTests

# Run all PetStore API tests
mvn test -pl api -am

# Generate Allure HTML report
mvn allure:report -pl api
open api/target/site/allure-maven-plugin/index.html
```

### Run against a different environment

```bash
BASE_URL=https://staging.myapi.com mvn test -pl api -am
```

## Configuration

All settings are controlled by environment variables (highest priority) or
`config/default.properties` (fallback). Never hard-code secrets.

| Variable | Default | Description |
|---|---|---|
| `BASE_URL` | `https://petstore.swagger.io/v2` | API base URL |
| `REQUEST_TIMEOUT_MS` | `10000` | HTTP timeout in ms |
| `RESPONSE_TIME_SLA_MS` | `3000` | Max acceptable response time |
| `MAX_RETRIES` | `2` | Retries on 5xx |
| `API_KEY` | _(blank)_ | API key header value |
| `BEARER_TOKEN` | _(blank)_ | OAuth bearer token |
| `LOG_ALL_REQUESTS` | `false` | Log full req/resp to console |
| `LLM_API_KEY` | _(blank)_ | Key for AI test generator |
| `LLM_MODEL` | `claude-sonnet-4-20250514` | LLM model |

## AI test generation

The `TestGenerator` class reads an OpenAPI spec and writes Java test classes automatically.

```bash
# Set your LLM API key
export LLM_API_KEY=your_key_here

# Run the generator against the PetStore spec
mvn exec:java \
  -pl shared \
  -Dexec.mainClass="com.aitesting.shared.ai.TestGeneratorRunner" \
  -Dexec.args="api/src/test/resources/petstore-openapi.json petstore"
```

Generated files land in `api/src/test/java/com/aitesting/api/aigenerated/petstore/`.

## CI/CD

Every push to `main` or `develop`:
1. Builds the `shared` library
2. Runs all API tests in parallel (3 threads)
3. Generates an Allure HTML report
4. Publishes the report to GitHub Pages
5. Uploads artifacts for 30 days

Allure report: `https://njmarshall.github.io/ai-test-automation/allure-report/`

## Shared library usage

Any new capstone project in `api/` gets the full framework for free:

```java
// One import for HTTP
import com.aitesting.shared.http.ApiClient;

// One import for assertions
import com.aitesting.shared.assertions.ResponseValidator;

// One import for test data
import com.aitesting.shared.dataprovider.TestDataFactory;

// Usage in a test
Response response = ApiClient.post("/quote", TestDataFactory.randomPetPayload());
ResponseValidator.from(response)
    .statusCode(201)
    .withinSla()
    .hasField("quoteId");
```
